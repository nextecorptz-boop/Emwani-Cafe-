Section 03

Scroll Timing Blueprint

Viewport heights represent total scroll travel, including GSAP-inserted pin spacers. "Pinned" sections use end: "+=Xvh" expressed as a percentage of window.innerHeight. All values assume desktop viewport ≥ 900px tall.

ChapterSection IDDurationGSAP ModeReasoning
00 Hero#hero100vhCSS parallax onlyNo GSAP pin needed. Background drifts via background-position on scroll event listener.
01 Origin#origin120vhIO stagger onlyExtra 20vh gives stagger cards time to complete reveal before user scrolls past.
02 Harvest#harvest180vh pinnedGSAP pin3-beat content reveal needs time. 180vh at typical scroll speed ≈ 9 seconds dwell.
03 Cherry Selection#cherry-selection120vhCSS scale + IOQuick chapter. Image zoom on enter, short copy. No pin justified.
04 Processing#processing180vhGSAP scrub (existing)Horizontal pan from start:"top bottom" to end:"bottom top" — adjust container to +=180vh.
05 Roasting#roasting200vh pinnedGSAP pin200vh (not 180vh) — fire/smoke imagery needs longer dwell to feel oppressive and cinematic.
06 Grinding#grinding160vh pinnedGSAP pinParticle loop needs 160vh to feel resolved before next chapter begins.
07 Brewing#brewing180vh pinnedGSAP pinBackground color tween needs full 180vh to feel gradual rather than snapping.
08 Assembly#assembly240vh pinnedGSAP pin + video scrubLongest pin. 240vh ≈ 8s video at typical scroll speed. video.currentTime = progress × duration.
09 Products#products160vhGSAP stagger (existing)Cards stagger at 75% viewport entry. Extra 60vh gives hover dwell time before scroll continues.
10 Café#cafe120vhIO revealTwo-column reveal. No pin needed.
11 Founder#founder100vhIO staggerGrid stagger left-right. Clean single viewport.
12 Contact#contact100vhIO staggerLast section. No scrolling past.
Total nominal scroll height: ~1,940vh (sum of pinned + unpinned sections)
With GSAP pin spacers, actual document height = section content height + all pin spacers. GSAP inserts these automatically — do not add manual spacer divs.

Why 240vh on Assembly (not less)

At typical laptop trackpad speed (~150px/s), 240vh on a 900px viewport = approximately 14–16 seconds of physical scroll to traverse the full 8-second video scrub. This is intentionally slow. It gives the viewer time to absorb each transformation beat — cherry, bean, roast, grind, filter, pour, steam, product. Rushing this section destroys its purpose.

Why 200vh on Roasting (not 180vh)

Steam and fire imagery needs longer dwell to build atmospheric tension. 180vh felt rushed in comparable luxury coffee and spirits implementations. The extra 20vh costs negligible scroll effort but significantly improves mood.

Mobile Behavior — All Pinned Sections

All GSAP pins are disabled on mobile (!isMobile guard in cinematic.js). On mobile, users see content revealed section by section via IntersectionObserver. Pinned-section timing becomes irrelevant — each section reverts to min-height: auto with normal padding. No GSAP spacers are inserted, so page height is dramatically shorter on mobile. This is correct behavior.

GSAP Implementation Pattern

// Pinned sections — update cinematic.js GSAP block
const pinnedSections = [
  { id: "#harvest",  end: "+=180%" },
  { id: "#roasting", end: "+=200%" },
  { id: "#grinding", end: "+=160%" },
  { id: "#brewing",  end: "+=180%" },
  { id: "#assembly", end: "+=240%" }
];

pinnedSections.forEach(({ id, end }) => {
  ScrollTrigger.create({
    trigger: id,
    start: "top top",
    end: end,
    pin: true,
    anticipatePin: 1
  });
});

// Assembly video scrub
const assemblyVideo = document.querySelector('#assembly video');
if (assemblyVideo) {
  assemblyVideo.addEventListener('loadedmetadata', () => {
    ScrollTrigger.create({
      trigger: '#assembly',
      start: 'top top',
      end: '+=240%',
      scrub: true,
      onUpdate: (self) => {
        assemblyVideo.currentTime = self.progress * assemblyVideo.duration;
      }
    });
  });
}

Each asset type has one best tool, one acceptable alternative, and one tool to avoid. Rationale is included so you can make judgment calls when circumstances change.

Best

OK
Adobe Firefly 3Good for editorial photography style. Less consistent landscape depth and atmospheric haze compared to MJ v7.
Avoid

Cherry-to-Bean Animation

Best
After Effects + Bodymovin + LottieFilesFull control over organic split motion, timing curves, and color shift. Produces ≤80KB Lottie JSON. Scrubs with ScrollTrigger via lottie.goToAndStop(progress * totalFrames, true).
OK
Rive (rive.app)Real-time interactive, smaller than Lottie, but steep learning curve for shape morphing on organic forms.
Avoid
Runway / KlingWould produce a video 5–15× larger file for the same visual. Video cannot be frame-accurately scrubbed at low scroll speeds.

Roasting Smoke Loop

Best
Runway Gen-3 AlphaBest fluid smoke simulation from text prompt. Loop mode stabilizes seamlessly. Enable slow-motion + 1080p output.
OK
Kling 1.6Strong motion quality. Slightly less reliable loop seam — may need 0.5s crossfade trim in AE.
Avoid
Pika 1.5Pika 1.5 struggles with abstract particle and fluid generation. Produces over-stylized, cartoon-adjacent smoke. Better suited for character motion.

Grinding Particles Loop

Best
After Effects + Trapcode ParticularPrecise physical simulation of dry particle behavior. Renders to transparent background for compositing over any dark section. Full control over gravity, turbulence, and color randomization.
OK
Runway Gen-3Faster. Prompt-driven. Less precise particle physics but acceptable for atmosphere-only use where exact grain size doesn't matter.
Avoid
SplineSpline particles are 3D interactive WebGL — not film-quality 2D particle renders. Adds ~500KB runtime overhead for a background loop.

Brewing Drip Loop

Best
Runway Gen-3 (slow motion)Outstanding fluid dynamics for liquid pour. Slow-motion mode produces the cinematic drip quality needed. Set camera motion to locked static.
OK
Kling 1.6Good fluid dynamics but slightly more stylized. May require color correction to match Emwani's warm amber palette.
Avoid
Gemini VeoCapable model but less proven specifically for macro liquid physics at slow-motion quality. Use for landscapes, not pour-overs.

Product Assembly Loop (Master Sequence)

Best
After Effects (full pipeline)Only tool that gives complete control over all 8 distinct beats, timing, and the product reveal at the end. Individual organic segments (pour, steam) can be generated in Runway and composited in AE or Premiere.
OK
After Effects segments + Runway for organic momentsHybrid approach: AE for structure and timing, Runway for the pour and steam segments specifically. Composite in Premiere Pro.
Avoid
BlenderOverkill. No 3D asset modeling is required. Blender would add production delay without quality gain for this 2D-compositing job.
