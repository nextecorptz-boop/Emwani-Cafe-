3 · Recommended HTML Structure

Reusable Video Panel Pattern

<figure class="cinematic-video reveal" data-cinematic-video>
  <video
    class="cinematic-video__el"
    poster="assets/story/posters/grinding-assembly-poster.webp"
    muted
    loop
    playsinline
    preload="none"
    aria-label="Coffee grinder in motion — silent loop">
    <source data-src="assets/story/video/grinding-assembly-loop.webm" type="video/webm">
  </video>
  <img
    class="cinematic-video__poster-fallback"
    src="assets/story/posters/grinding-assembly-poster.webp"
    alt="Coffee grinder mid-pour, frozen frame"
    loading="lazy"
    width="1600" height="900">
</figure>

Why this shape:

  • <source data-src> defers the actual download until the JS observer attaches it — true lazy load.
  • The sibling <img> is the safety net: it shows immediately, and stays under the video. If the video element fails or is suppressed (reduced-motion / no JS), the poster is what the viewer sees.
  • preload="none" + poster attribute prevents the blank-frame flash even before JS fires.
  • No audio source means there is nothing to mute incorrectly.

Add to assets/css/cinematic.css — do not modify product styles.

/* === Cinematic Video Panel === */
.cinematic-video {
  position: relative;
  width: 100%;
  max-width: 1100px;
  margin: var(--space-8) auto 0;
  aspect-ratio: 16 / 9;
  border-radius: var(--r-lg);
  overflow: hidden;
  box-shadow: var(--shadow-cinema);
  background: var(--char-1000); /* prevents white flash before poster paints */
}

.cinematic-video__el,
.cinematic-video__poster-fallback {
  position: absolute;
  inset: 0;
  width: 100%;
  height: 100%;
  object-fit: cover;
  display: block;
}

/* Poster sits BELOW the video. Video paints over it once playing. */
.cinematic-video__poster-fallback { z-index: 1; }
.cinematic-video__el              { z-index: 2; }

/* If JS marks the panel as "static" (reduced motion, mobile-skip, or load failure),
   hide the video element and let the poster show through. */
.cinematic-video.is-static .cinematic-video__el { display: none; }

/* Optional product backdrop variant — clip C only */
.cinematic-video--backdrop {
  position: absolute;
  inset: 0;
  margin: 0;
  max-width: none;
  border-radius: 0;
  box-shadow: none;
  opacity: 0.18;
  pointer-events: none;
  z-index: 0;
}

@media (max-width: 768px) {
  .cinematic-video { margin-top: var(--space-6); border-radius: var(--r-md); }
}

@media (prefers-reduced-motion: reduce) {
  .cinematic-video__el { display: none !important; }
}

Notes:

  • aspect-ratio: 16/9 reserves layout space → zero CLS.
  • object-fit: cover preserves cinematic framing across breakpoints.
  • No position: fixed, no horizontal translate — guarantees no horizontal scroll.
  • Reduced-motion is enforced in CSS as a backstop even if JS misfires.

5 · JS Architecture for Safe Video Loading / Play-Pause

Append to assets/js/cinematic.js, after the existing IntersectionObserver block. Self-contained, additive, no rewrites.

// --- Phase 6: Cinematic Video Panels ---
(function initCinematicVideos() {
  const panels = document.querySelectorAll('[data-cinematic-video]');
  if (!panels.length) return;

  const reduce  = window.matchMedia('(prefers-reduced-motion: reduce)').matches;
  const isMobile = window.matchMedia('(max-width: 768px)').matches;

  panels.forEach(panel => {
    const video = panel.querySelector('video');
    if (!video) return;

    // Reduced motion OR mobile → poster only, never load the clip
    if (reduce || isMobile) {
      panel.classList.add('is-static');
      return;
    }

    // Promote data-src → src lazily when the panel enters view
    const source = video.querySelector('source[data-src]');
    let loaded = false;

    const load = () => {
      if (loaded) return;
      loaded = true;
      if (source) {
        source.src = source.dataset.src;
        video.load();
      }
    };

    // Fallback to poster on any load/decode error
    video.addEventListener('error', () => panel.classList.add('is-static'), { once: true });

    const io = new IntersectionObserver((entries) => {
      entries.forEach(entry => {
        if (entry.isIntersecting) {
          load();
          const p = video.play();
          if (p && typeof p.catch === 'function') {
            p.catch(() => panel.classList.add('is-static'));
          }
        } else {
          if (!video.paused) video.pause();
        }
      });
    }, { threshold: 0.35 });

    io.observe(panel);
  });
})();

Guarantees:

  • Will not crash if zero panels exist (early return).
  • Will not download video bytes on mobile or reduced-motion users.
  • Pauses when offscreen → no background CPU/GPU drain during scroll.
  • play() promise rejection (autoplay-blocked browsers) gracefully falls back to poster.
  • Decode/network errors flip .is-static → poster takes over.

6 · Reduced-Motion Fallback

  • JS layer: matchMedia('(prefers-reduced-motion: reduce)') short-circuits — no src assignment, no play() call, no network request for the video file.
  • CSS layer: @media (prefers-reduced-motion: reduce) { .cinematic-video__el { display: none !important; } } guarantees that even if JS executes, the video element is invisible.
  • Poster layer: The sibling <img> remains visible at all times; the design works as a still composition.

7 · Mobile Fallback

  • Below 768px the panel is marked .is-static at init — video element is hidden, poster image carries the section.
  • No autoplay on mobile data plans; respects user bandwidth.
  • No GSAP pin or scrub interaction is added on mobile (existing !isMobile guard at cinematic.js:16 remains the source of truth for pinned sections).
  • Posters are sized to the same aspect-ratio: 16/9 container, so layout and reveal timing are identical to desktop.

10 · QA Checklist

Functional

  • Both required .webm files return HTTP 200.
  • Both poster .webp files return HTTP 200.
  • No 404 in DevTools Network tab on full-page scroll.
  • Video plays only when its panel is ≥ 35 % visible.
  • Video pauses when scrolled offscreen (verify in Network/Performance tab — no continued decode).
  • Video has no audio track (verify with ffprobe or by muting system).
  • All <video> elements carry muted loop playsinline preload="none".
  • Disabling JavaScript still shows the poster image cleanly.
  • Forcing the video URL to 404 still shows poster (error fallback path).

Reduced Motion

  • OS-level reduced motion → no video element visible, poster shown.
  • No video network request initiated under reduced motion (Network tab confirms).

Mobile (≤ 768px)

  • No video file requested on mobile (Network tab on iPhone SE / 375 px).
  • Poster fills the panel cleanly at the same aspect ratio.
  • No horizontal scroll at 320, 375, 414, 768 px widths.
  • Existing GSAP pins remain disabled on mobile.

Layout / CLS

  • Lighthouse CLS < 0.1 for the page.
  • No section above the panel jumps when the video swaps in.
  • Existing #brewing dark→cream tween still resolves correctly with the video present.

Brand / Story

  • Footer Nextec disclaimer remains intact and visible.
  • Product section visually unchanged (clip C either omitted or proven non-disruptive).
